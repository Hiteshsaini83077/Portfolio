Project One: Bitcoin Price Prediction

Built a machine learning model to forecast Bitcoin prices using LSTM, regression and time-series analysis.

Python Pandas NumPy Matplotlib Scikit-learn TensorFlow Keras

Overview:
This project focuses on building a predictive model to forecast Bitcoin prices using historical data and machine learning techniques. The primary goal is to help investors, traders, and researchers better understand market trends and make informed decisions by leveraging data-driven insights.

Key Features:

  • Collected and preprocessed historical Bitcoin price data (OHLC, volume, etc.)
  • Applied data visualization to analyze trends, volatility, and correlation
  • Implemented multiple machine learning models, including:
    • Linear Regression
    • LSTM (Long Short-Term Memory)
    • Random Forest Regressor
  • Evaluated model performance using metrics like RMSE, MAE, and R² score
  • Visualized predictions vs. actual price using interactive plots
  • Explored time series analysis and future price forecasting

Challenges Solved:
  • Addressed non-stationary data through differencing and normalization
  • Managed missing values and outliers in financial time series
  • Tuned hyperparameters to reduce overfitting in neural networks

Outcome:
Successfully developed a robust model that provides short-term predictions for Bitcoin prices with reasonable accuracy. The project demonstrates the practical use of AI in financial markets and highlights the challenges of predicting highly volatile assets.
GitHub Repository: https://github.com/Hiteshsaini83077/Bitcoin_prediction

Project two : Creating GLOB library(file handling library in Python)

Designed and developed a custom file-handling library using Python that mimics the functionality of the built-in glob module. The library allows pattern-based file searching using wildcards (*, ?, []) and recursive directory traversal, enabling efficient filtering of filenames within a given directory tree.

Python OS module fnmatch module

Overview:
Designed and developed a custom file-handling library using Python that mimics the functionality of the built-in glob module. The library allows pattern-based file searching using wildcards (*, ?, []) and recursive directory traversal, enabling efficient filtering of filenames within a given directory tree.

Key Features:

  • Custom implementation of wildcard matching (*.txt, file_?.py, etc.)
  • Support for recursive file search across nested folders
  • Clean and modular code structure for reusability
  • Extensive testing with various directory structures and file types

Challenges faced:
  • Building a pattern-matching engine from scratch without relying on Python's standard glob or fnmatch modules
  • Handling edge cases like hidden files, symbolic links, and deeply nested directories
  • Ensuring performance and accuracy for large file systems
GitHub Repository: https://github.com/Hiteshsaini83077/PythonGlobLibrary

Project Four : Agricultural Crop Diseases analysis

Industrial Hackathon Project

This project is about the crop disese analysis

Python TensorFlow Keras image processing OpenCV Flask HTML CSS

Description:
Developed a web-based application that allows users—especially farmers—to upload photos of crop leaves for automated disease analysis. The system identifies visible symptoms of diseases such as leaf blight, mildew, and fungal infections, and provides basic guidance or warnings. This project aims to support smarter agriculture through early disease detection using computer vision.

Key Features:

  • Implemented image processing techniques to analyze visual symptoms on leaf surfaces.
  • Developed a user-friendly upload interface for users to submit images.
  • Applied classification algorithms to categorize diseases based on leaf image patterns.
  • Generated informative outputs, including disease names and possible causes or remedies.

Challenges faced:
  • Collecting a reliable dataset of diverse crop diseases with clear symptoms.
  • Ensuring accuracy of analysis under different lighting and image quality conditions.
  • Building a responsive and intuitive front end for farmers with little technical background.
GitHub Repository: https://github.com/Hiteshsaini83077/Agricultural_project
